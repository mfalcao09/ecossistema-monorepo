✗

Concierge IA

✗

Revenue Intelligence

A Visão: Living Intelligence Hub

Cada interação do cliente gera insights que alimentam o sistema, criando um ciclo virtuoso de personalização, retenção e monetização

🤖
Filosofia IA-Native: A IA não é um botão que o usuário clica. A IA é o sistema operacional do relacionamento. Ela observa, aprende, antecipa e age — o humano supervisiona, ajusta e humaniza. O resultado: a imobiliária que "lê a mente" do cliente.
P1

Onboarding InteligentePrimeira impressão perfeita, personalizada por IA

AI Core Diferenciador

F1 — Perfil Comportamental Dinâmico (DNA do Cliente)

Micro-quiz gamificado (2 min) no primeiro acesso que constrói o perfil comportamental do cliente usando IA. Determina: estilo de comunicação preferido (texto/áudio/chamada), nível de autonomia digital, perfil emocional (analítico, relacional, pragmático), e expectativas com o imóvel.

  • Gemini analisa padrões de resposta para gerar "DNA do Cliente"
  • Perfil influencia TODA comunicação futura (tom, canal, frequência)
  • Atualização contínua: IA refina o perfil a cada interação
  • Edge Function: relationship-client-dna
  • Tabela: client_behavioral_profiles
Frontend + AI Alto Impacto

F2 — Jornada de Boas-Vindas Adaptativa

Tour personalizado que se adapta em tempo real ao perfil do cliente. Cliente primeiro imóvel? Tour detalhado sobre garantias, seguros, manutenção. Cliente investidor? Foco em dashboards financeiros e performance.

  • Steps dinâmicos gerados pela IA com base no DNA do Cliente
  • Checklist interativo: vistoria, documentos, apresentação do time
  • Marco "Ativação": IA detecta quando cliente completou setup mínimo
  • Notificação proativa: "Falta só configurar seu seguro incêndio!"
  • Componente: AdaptiveOnboardingWizard.tsx
AI Core

F3 — Sentiment Scanner de Primeiro Contato

Análise de sentimento automática nas primeiras interações (tickets, mensagens, emails). Detecta sinais de frustração precoce ("ninguém me ligou", "estou esperando desde...") e adapta prioridade de atendimento.

  • Classificação: Entusiasmado / Neutro / Preocupado / Frustrado
  • Auto-escalation se sentimento negativo nas primeiras 48h
  • Alimenta o Health Score desde o dia 1
  • Edge Function: relationship-sentiment-analyzer
P2

Portal do Cliente com Concierge IAO cliente se sente ouvido, entendido e cuidado — 24/7

AI Core Diferenciador #1

F4 — IntelliHome: Concierge Multimodal

Assistente IA que conversa com o cliente de verdade — não é FAQ bot. Entende contexto ("minha torneira está vazando de novo"), tem memória persistente ("você reportou isso em janeiro"), e toma ações (abre ticket, agenda vistoria, consulta contrato).

  • RAG com base de conhecimento: contratos, manuais, histórico do imóvel
  • Memória contextual: lembra interações passadas e preferências
  • Ações reais: abre tickets, agenda vistorias, envia 2ª via de boleto
  • Multicanal: funciona no portal web, WhatsApp e email
  • Handoff inteligente: escala para humano quando necessário, com contexto completo
  • Edge Function: relationship-concierge-ai (com Function Calling)
  • Tabelas: concierge_conversations, concierge_memory
Frontend Alto Impacto

F5 — Digital Twin do Imóvel

Timeline visual do imóvel com histórico completo construído automaticamente por IA. Manutenções realizadas, vistorias, ocorrências, modificações. O cliente "pergunta ao seu imóvel": "Quando foi a última manutenção do ar-condicionado?"

  • Timeline auto-gerada a partir de tickets, vistorias, manutenções
  • Chat contextual vinculado ao imóvel específico
  • Alertas proativos: "Seu ar-condicionado tem 11 meses sem manutenção"
  • Documentos do imóvel: contrato, vistoria, manual do condomínio
  • Componente: PropertyDigitalTwin.tsx
AI Core Diferenciador

F6 — Proactive Life Events Engine

IA detecta "momentos de vida" e antecipa necessidades do cliente antes mesmo dele perceber. Aniversário do contrato? Mensagem personalizada + oferta de renovação. Cliente completou 1 ano? Relatório de valorização do imóvel.

  • Calendário inteligente: datas contratuais, aniversários, vencimentos
  • Pattern matching: "Cliente pagou 3 meses adiantado = estabilidade financeira = propensão a upgrade"
  • Triggers contextuais: reajuste IGPM, mudança de mercado, fim de garantia
  • Geração automática de conteúdo personalizado para cada momento
  • Edge Function: relationship-life-events-engine
  • Tabela: client_life_events
P3

Churn Radar 360° + InterceptorAntecipar problemas antes do cliente reclamar

AI Core Diferenciador

F7 — Churn Prediction Engine Multi-Dimensional

Modelo preditivo que vai muito além de "atraso no pagamento". Analisa 3 camadas de sinais: Quantitativos (frequência de tickets, atrasos, login no portal), Qualitativos (sentimento em mensagens, tempo de resposta, palavras de desengajamento), e Contextuais (mercado em alta, contrato vencendo, vizinho reclamou).

  • Score 0-100 com "Top 3 Razões" explicáveis em linguagem natural
  • Janelas de predição: 30, 60, 90 dias
  • Feature engineering específico para aluguel/venda imobiliária BR
  • Análise de sentimento integrada em todas as comunicações
  • Dashboard com mapa de calor de risco por carteira
  • Edge Function: relationship-churn-predictor
  • Tabela: churn_predictions, churn_signals
AI Core + Automação Alto Impacto

F8 — Churn Interceptor: Salvamento Automático

Quando Churn Score ultrapassa threshold, ativa protocolo automatizado de retenção, gerando script personalizado para o CS, sugerindo ofertas de retenção calibradas ao perfil do cliente, e escalando para gerência em casos críticos.

  • Score > 50: Alerta passivo + sugestão de ação para CS
  • Score > 70: Protocolo ativo — task automática + script IA personalizado
  • Score > 85: Escalation gerencial + oferta especial (desconto, upgrade, flexibilidade)
  • Cada intervenção alimenta modelo de "o que funciona" (feedback loop)
  • ROI calculado automaticamente: custo retenção vs. custo de novo cliente
  • Edge Function: relationship-churn-interceptor
P4

Offboarding HumanizadoTransformar saída em oportunidade e aprendizado

AI Core

F9 — Exit Experience Architecture

Fluxo de saída conduzido por IA que dignifica o cliente, coleta inteligência valiosa, e mantém a porta aberta. Entrevista de saída conversacional, transição suave de documentos, e programa "Amigos Intentus" para indicações futuras.

  • Exit Interview IA: conversa natural que extrai razões reais da saída
  • Classificação automática: Preço / Qualidade / Mudança de cidade / Compra própria / Outro
  • Pacote de transição: histórico do imóvel, documentos, timeline exportável
  • "Adeus, mas não tanto": oferta de notificações sobre novos imóveis na região desejada
  • Programa de indicação: "Indique um amigo e ganhe benefícios no próximo aluguel"
  • Edge Function: relationship-exit-intelligence
AI Core Alto Impacto

F10 — Feedback Intelligence Loop

IA que transforma TODOS os feedbacks (NPS, exit interviews, tickets, mensagens) em inteligência acionável. Agrupa padrões, identifica tendências, gera relatórios de product insights, e fecha o ciclo notificando clientes quando seus feedbacks geram mudanças.

  • Clustering automático: "32 clientes mencionaram demora em manutenção"
  • Trend detection: "Reclamações sobre condomínio aumentaram 40% em março"
  • Fechamento de loop: "Vimos seu feedback — implementamos X. Obrigado!"
  • Dashboard de Product Insights para tomada de decisão
  • Edge Function: relationship-feedback-intelligence
  • Tabela: feedback_intelligence, feedback_clusters
P5

Revenue Intelligence EngineMonetizar o relacionamento com inteligência e timing

AI Core Diferenciador

F11 — Next Best Action Engine

Engine IA que identifica o momento perfeito e a ação perfeita para cada cliente. Cross-sell de seguros para quem ainda não tem, upsell de imóvel maior para família crescendo, renovação antecipada com desconto para reter clientes de alto valor.

  • Modelo de propensão: probabilidade de aceitar cada tipo de oferta
  • Timing otimizado: "melhor momento para oferecer" baseado em comportamento
  • Personalização: oferta gerada pela IA com linguagem do perfil do cliente
  • A/B testing automático: IA testa variações e aprende o que converte
  • Dashboard de oportunidades com valor estimado por cliente
  • Edge Function: relationship-next-best-action
  • Tabelas: revenue_opportunities, nba_recommendations
AI Core + Integration Alto Impacto

F12 — Revenue Attribution & Lifetime Value Predictor

Modelo que calcula o LTV real de cada cliente e atribui receita a cada ação de relacionamento. Responde: "Quanto vale investir para reter este cliente?" e "Qual ação de CS gerou mais receita?"

  • LTV preditivo: valor esperado do cliente nos próximos 12/24/36 meses
  • Attribution model: qual touchpoint gerou renovação/upgrade/indicação
  • Segmentação por valor: Platinum / Gold / Silver / Bronze automática
  • ROI de CS: "Investimos R$ X em retenção, geramos R$ Y em renovações"
  • Alimenta decisões de alocação: mais CS para clientes de maior LTV
  • Edge Function: relationship-revenue-attribution
  • Tabela: client_ltv_predictions, revenue_attributions

Schema de Dados — Novas Tabelas

Estrutura de dados necessária para suportar as 12 features

-- PILAR 1: Onboarding CREATE TABLE client_behavioral_profiles ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), person_id UUID REFERENCES people(id), communication_style TEXT CHECK (communication_style IN ('text','audio','video','call')), autonomy_level TEXT CHECK (autonomy_level IN ('high','medium','low')), emotional_profile TEXT CHECK (emotional_profile IN ('analytical','relational','pragmatic')), experience_level TEXT CHECK (experience_level IN ('first_time','experienced','investor')), dna_raw JSONB, -- Full quiz responses + AI analysis ai_summary TEXT, -- Human-readable profile summary last_refined_at TIMESTAMPTZ, -- When AI last updated the profile created_at TIMESTAMPTZ DEFAULT now() ); -- PILAR 2: Portal do Cliente CREATE TABLE concierge_conversations ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), person_id UUID REFERENCES people(id), channel TEXT CHECK (channel IN ('portal','whatsapp','email')), messages JSONB[], -- Array of {role, content, timestamp, actions_taken} context JSONB, -- Contract, property, recent tickets resolved BOOLEAN DEFAULT false, escalated_to UUID REFERENCES profiles(id), created_at TIMESTAMPTZ DEFAULT now() ); CREATE TABLE concierge_memory ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), person_id UUID REFERENCES people(id), memory_type TEXT CHECK (memory_type IN ('preference','issue','interaction','context')), content TEXT, importance_score INT DEFAULT 5, expires_at TIMESTAMPTZ, created_at TIMESTAMPTZ DEFAULT now() ); CREATE TABLE client_life_events ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), person_id UUID REFERENCES people(id), event_type TEXT, -- contract_anniversary, renewal_window, guarantee_expiry, etc. event_date DATE, action_taken TEXT, action_result TEXT, ai_recommendation JSONB, created_at TIMESTAMPTZ DEFAULT now() ); -- PILAR 3: Churn Prediction CREATE TABLE churn_predictions ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), person_id UUID REFERENCES people(id), contract_id UUID REFERENCES contracts(id), score INT CHECK (score BETWEEN 0 AND 100), prediction_window INT, -- 30, 60, 90 days top_reasons JSONB, -- [{reason, weight, explanation}] recommended_actions JSONB, -- [{action, priority, script}] model_version TEXT, predicted_at TIMESTAMPTZ DEFAULT now() ); CREATE TABLE churn_signals ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), person_id UUID REFERENCES people(id), signal_type TEXT CHECK (signal_type IN ('quantitative','qualitative','contextual')), signal_name TEXT, signal_value NUMERIC, raw_data JSONB, detected_at TIMESTAMPTZ DEFAULT now() ); CREATE TABLE churn_interventions ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), prediction_id UUID REFERENCES churn_predictions(id), intervention_type TEXT, -- contact, offer, escalation intervention_detail JSONB, outcome TEXT CHECK (outcome IN ('retained','churned','pending')), executed_by UUID REFERENCES profiles(id), executed_at TIMESTAMPTZ, created_at TIMESTAMPTZ DEFAULT now() ); -- PILAR 4: Offboarding CREATE TABLE exit_interviews ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), person_id UUID REFERENCES people(id), contract_id UUID REFERENCES contracts(id), conversation JSONB, -- Full AI interview transcript primary_reason TEXT, -- price, quality, relocation, purchase, other secondary_reasons TEXT[], sentiment_score INT, ai_insights JSONB, referral_offered BOOLEAN DEFAULT false, referral_accepted BOOLEAN DEFAULT false, created_at TIMESTAMPTZ DEFAULT now() ); CREATE TABLE feedback_clusters ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), cluster_name TEXT, cluster_theme TEXT, feedback_count INT, avg_sentiment NUMERIC, sample_quotes JSONB, trend TEXT CHECK (trend IN ('rising','stable','declining')), action_taken TEXT, action_communicated BOOLEAN DEFAULT false, detected_at TIMESTAMPTZ DEFAULT now() ); -- PILAR 5: Revenue Intelligence CREATE TABLE revenue_opportunities ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), person_id UUID REFERENCES people(id), opportunity_type TEXT CHECK (opportunity_type IN ('cross_sell','upsell','renewal','referral')), product TEXT, -- insurance, upgrade, renewal, etc. estimated_value NUMERIC, propensity_score INT CHECK (propensity_score BETWEEN 0 AND 100), optimal_timing DATE, personalized_pitch TEXT, -- AI-generated offer text status TEXT CHECK (status IN ('identified','presented','accepted','declined','expired')), created_at TIMESTAMPTZ DEFAULT now() ); CREATE TABLE nba_recommendations ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), person_id UUID REFERENCES people(id), action_type TEXT, action_detail JSONB, priority INT, reason TEXT, presented_at TIMESTAMPTZ, outcome TEXT, created_at TIMESTAMPTZ DEFAULT now() ); CREATE TABLE client_ltv_predictions ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), person_id UUID REFERENCES people(id), ltv_12m NUMERIC, ltv_24m NUMERIC, ltv_36m NUMERIC, segment TEXT CHECK (segment IN ('platinum','gold','silver','bronze')), confidence NUMERIC, model_version TEXT, predicted_at TIMESTAMPTZ DEFAULT now() ); CREATE TABLE revenue_attributions ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID REFERENCES tenants(id), revenue_event TEXT, -- renewal, upgrade, cross_sell, referral revenue_value NUMERIC, attributed_to TEXT, -- touchpoint that caused conversion person_id UUID REFERENCES people(id), contract_id UUID REFERENCES contracts(id), attribution_model TEXT, -- first_touch, last_touch, multi_touch created_at TIMESTAMPTZ DEFAULT now() );
