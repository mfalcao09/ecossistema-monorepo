🏗️ Dimensão 1: Arquitetura & Backend

Crítica 1.1 — Middleware de Domínio com IP Fixo

Problema

`middleware.ts` hardcoda listas de domínios (`erp.fic`, `gestao.fic`, `diploma.fic`) com IPs fixos para bypass de rate limiting.

Impacto

Violação do princípio de least privilege. IPs podem mudar causando outage ou serem interceptados. Não há fallback dinâmico via DNS.

Diretriz de Solução

Migrar para Service Discovery (Consul, Eureka) ou DNS TXT records com TTL curto. Remover whitelist de IPs do código.

Média 1.2 — Padrão Single-Tenant Implícito

Problema

Arquitetura multi-tenant via `tenant_id`/`institution_id` mas sem isolamento de schema no Postgres (single database).

Impacto

Vazamento de dados cross-tenant via bugs em queries ou RLS mal configurada. Queries lentas afetam todos os tenants simultaneamente.

Diretriz de Solução

Considerar Postgres schemas separados por instituição ou migrate para multi-database pattern com tenant-sharding.

Média 1.3 — Webhook BRy Sem HMAC

Problema

`bry-assinatura-pdf/route.ts` não implementa validação HMAC-SHA256 como o webhook WhatsApp.

Impacto

Qualquer requisição autenticada na rede interna pode manipular status de assinaturas PDF. Sem garantia de integridade da origem.

Diretriz de Solução

Implementar `Bry-Webhook-Signature` header com HMAC-SHA256. Validar `X-Forwarded-For` para garantir origem via Load Balancer.

Média 1.4 — IP Validation Regex Permissivo

Problema

`rate-limit.ts` usa `/^[0-9a-fA-F:]+$/` para IPv6 — aceita qualquer string hexadecimal com `:`.

Impacto

Bypass de rate limiting via headers `X-Forwarded-For` forjados.

Diretriz de Solução

Usar `net.isIP()` do Node.js ou `is-ip` package. Validar IPv4 e IPv6 separadamente.

Baixa 1.5 — Fallback Memória no Rate Limiter

Problema

`rate-limit.ts` usa `new Map()` em memória quando Redis falha — sem TTL cleanup.

Impacto

Memory leak em ambientes com alto uptime. Rate limiting pode ser completamente ineffective.

Diretriz de Solução

Implementar `setInterval` para cleanup periódico ou usar `lru-cache` com `maxAge`.

Média 1.6 — In-Memory Filtering em Diplomas

Problema

`GET /api/diplomas` faz filter by search in-memory via `filter()`.

Impacto

Não escala. Carrega TODOS os diplomas filtrados do Postgres para memória antes de paginar.

Diretriz de Solução

Migrar filter para SQL `WHERE` clause com `tsvector` (Full-Text Search) ou `ILIKE` indexado.

📝 Dimensão 2: Code Quality & Language

Crítica 2.1 — Diploma Detail Page (108KB, 2410 linhas)

Problema

`page.tsx` com 2410 linhas é um God Component que viola SRP, OCP e quase todos os princípios SOLID.

Impacto

Impossível manter, testar ou fazer code review. Qualquer change é risco de regressão. Bug #H modal embutido no mesmo arquivo.

Diretriz de Solução

Extrair para:
• `components/DiplomaTabs/` (Dados, XMLs, Documentos, Acervo, Histórico)
• `hooks/useDiplomaWorkflow.ts`
• `components/DiplomaActions.tsx`
• `components/PipelineStatus.tsx`

Média 2.2 — API Route Monolítica

Problema

`diplomas/[id]/route.ts` (325 linhas) acumulou múltiplas responsabilidades: GET, DELETE, PATCH, field mapping, extra queries.

Impacto

Violação do Princípio da Responsabilidade Única. Dificulta caching granular e testing isolado.

Diretriz de Solução

Split em services separados: `GetDiplomaService`, `UpdateDiplomaService`, `DeleteDiplomaService`.

Baixa 2.3 — Async Fire-and-Forget Inconsistente

Problema

`cadeia-custodia.ts` tem `registrarCustodiaAsync` mas `document-converter` usa callback pattern diferente.

Impacto

Inconsistência de error handling. Async fire-and-forget pode silently fail.

Diretriz de Solução

Padronizar para `queue.add()` (Bull/BullMQ) ou Domain Events com transactional outbox pattern.

Baixa 2.4 — Magic Numbers Sem Constantes

Problema

Valores como `2h`, `100`, `5s`, `30_000` espalhados no código sem documentação.

Impacto

Configuração oculta. Changes acidentais podem causar outages silenciosos.

Diretriz de Solução

Extrair para `src/config/constants.ts` com JSDoc explicando rationale de cada valor.

Média 2.5 — Error Handling Inconsistente

Problema

APIs retornam formatos variados: `{ error: string }`, `throw new Error()`, `NextResponse.error()`.

Impacto

Frontend precisa tratar múltiplos formatos de erro. Logging inconsistente.

Diretriz de Solução

Criar `ApiResponse` helper com métodos `success()`, `error()`, `validation()`.

🎨 Dimensão 3: Frontend & UI/UX

Média 3.1 — Portal Client (1020 linhas)

Problema

`PortalPageClient.tsx` mistura concerns: Turnstile, dual-tab UI, RVDD viewer, XML validator, API calls.

Impacto

Loading states complexos com `isInitialLoad` ref pattern. Dificulta SSR/SSG.

Diretriz de Solução

Extrair: `PortalSearchForm.tsx`, `RvddViewer.tsx`, `XmlValidator.tsx`, `useDiplomaPortal.ts`.

Média 3.2 — XML Validator No Client Side

Problema

XML validator roda no browser sem XSD schema caching. Qualquer browser refresh recarrega schemas.

Impacto

Performance degrade com múltiplos uploads. Sem caching de schemas MEC.

Diretriz de Solução

Implementar React Query com `staleTime: Infinity` para schemas XSD. Considerar server-side validation com Redis cache.

Baixa 3.3 — RVDD Proxy Sem Cache

Problema

`/api/portal/rvdd-proxy` faz proxy de PDFs sem cache.

Impacto

Cada visualização baixa o PDF novamente. Custo de egress elevado + latency.

Diretriz de Solução

Implementar `Cache-Control: public, max-age=31536000`. Usar Cloudflare Cache Rules.

Baixa 3.4 — Override Modal Sem Confirmation

Problema

Modal de override para violações de business rules (Bug #H) não pede confirmação explícita.

Impacto

Usuário pode clicar errado e circumvent regras de compliance.

Diretriz de Solução

Adicionar step de confirmação com texto digitado ("TENHO CIÊNCIA") ou double-click pattern.

Média 3.5 — Loading States Não Granulares

Problema

`page.tsx` usa `isInitialLoad` ref pattern genérico — não distingue refresh de skeleton initial load.

Impacto

UX inconsistente. Loading spinner aparece desnecessariamente.

Diretriz de Solução

Normalizar para TanStack Query com `isFetching` vs `isLoading` states.

⚡ Dimensão 4: Performance & Optimization

Crítica 4.1 — N+1 Queries no Diploma Detail

Problema

`diplomas/[id]/route.ts` faz queries separadas para `xml_gerados`, `extracao_sessoes`, `fluxo_assinaturas`.

Impacto

4 round-trips ao banco por cada acesso. 400 concurrent users = 1600 queries/segundo.

Diretriz de Solução

Usar JOIN com JSON aggregation:

SELECT d.*, 
  json_agg(xml) as xmls,
  json_agg(sessao) as sessoes
FROM diplomas d
LEFT JOIN xml_gerados xml ON xml.diploma_id = d.id
WHERE d.id = $1
GROUP BY d.id

Média 4.2 — Document Converter Sem Concurrency Control Global

Problema

`executarComLimite(4)` local — cada instância Railway tem seu próprio limite.

Impacto

3 instâncias = 12 concurrent jobs. Sem distributed coordination em spike scenarios.

Diretriz de Solução

Implementar Upstash QStash ou Redis-based distributed semaphore.

Média 4.3 — Sem Gzip/Brotli Compression

Problema

Nenhum middleware de compression configurado em `next.config.mjs`.

Impacto

JSON responses não comprimidos. Mobile users experimentam latency.

Diretriz de Solução

Adicionar `compress` middleware (Express) ou `next-compose-plugins`.

Média 4.4 — Supabase Realtime Sem Subscription Cleanup

Problema

Componentes que usam `supabase.channel()` sem `unsubscribe()` em cleanup.

Impacto

WebSocket connections acumulam. Browser tab leak em SPAs.

Diretriz de Solução

Implementar useEffect com cleanup:

useEffect(() => {
  const channel = supabase.channel(...)
  return () => { channel.unsubscribe() }
}, [])

Baixa 4.5 — Batch Size Fixed em Migration

Problema

`pii-encryption.ts` usa `100` records/batch fixo. Não adaptativo a load.

Impacto

Horário de pico = lock contention. Off-peak = capacidade subaproveitada.

Diretriz de Solução

Implementar adaptive batching via `pg_stat_activity`. Usar BACKGROUND worker.

Baixa 4.6 — Sem Database Index em Filter Columns

Problema

Filtros por `nome`, `cpf`, `curso` em diplomas não têm índices explícitos.

Impacto

Table scans em `diplomas` com milhões de registros.

Diretriz de Solução

Adicionar índices parciais:

CREATE INDEX idx_diplomas_nome 
  ON diplomas(nome) WHERE deleted_at IS NULL;
CREATE INDEX idx_diplomas_cpf_hash 
  ON diplomas(cpf_hash) WHERE cpf_hash IS NOT NULL;

🔒 Dimensão 5: Security

Crítica 5.1 — CSRF Double Submit Sem HttpOnly

Problema

`csrf.ts` stores CSRF token em cookie non-httpOnly (`_csrf`).

Impacto

XSS pode ler token CSRF via `document.cookie` e fazer requests cross-site. Se houver stored XSS, CSRF protection é bypassed.

Diretriz de Solução

Migrar para Same-Site Cookies (default browsers modernos) OU usar `CSRF_TOKEN` cookie HttpOnly + custom header.

Crítica 5.2 — BRy Webhook Sem HMAC Validation

Problema

`bry-assinatura-pdf/route.ts` não tem HMAC validation.

Impacto

Qualquer serviço interno pode POSTar falsificando status de assinatura.

Diretriz de Solução

Implementar `Bry-Signature` header com HMAC-SHA256. Log e reject requests sem signature.

Média 5.3 — Service Role Client Embutido em API Routes

Problema

`diplomas/[id]/route.ts` usa `createClient(url, serviceRoleKey)` diretamente na API route.

Impacto

Se API route for mal configurada, service role key pode vazar via stack traces ou logs.

Diretriz de Solução

Usar Server-side Supabase Client com JWT. Service role apenas em background jobs (RLS bypass é anti-pattern).

Média 5.4 — Rate Limit Bypass Via IPv6 Spoofing

Problema

`isValidIP` regex aceita strings forjadas em `X-Forwarded-For`.

Impacto

Atacante pode bypassear rate limiting forjando headers.

Diretriz de Solução

Validar IPs com `net.isIP()`. Trust apenas primeiro IP não-known-private.

Média 5.5 — Security Logger Sem Encryption

Problema

`security-logger.ts` logs eventos de segurança em plaintext.

Impacto

Se storage for comprometida, detalhes de ataques são expostos.

Diretriz de Solução

Criptografar campos `details` e `ip_address` com chave do Vault.

Média 5.6 — PII Encryption Sem Key Rotation

Problema

`pii-encryption.ts` usa `PII_ENCRYPTION_KEY` fixed. Não há mecanismo de rotação.

Impacto

Se chave for comprometida, todos os dados PII são decryptable. Não compliance com LGPD.

Diretriz de Solução

Implementar Envelope Encryption com KEK/DEK. Rotacionar KEK periodicamente via Vault.

Média 5.7 — Cadeia de Custódia Sem Distributed Consensus

Problema

`cadeia-custodia.ts` calcula `hash_anterior` localmente — não há Distributed Hash Chain.

Impacto

Em multi-instância, cada instância pode criar branch chains divergentes.

Diretriz de Solução

Implementar `SELECT FOR UPDATE SKIP LOCKED` ao ler `hash_anterior`.

Baixa 5.8 — Turnstile Timeout Não Configurado

Problema

`validar-codigo/route.ts` não tem timeout no fetch do Turnstile verification.

Impacto

Slow/failed Turnstile API pode causar request timeout.

Diretriz de Solução

Adicionar `AbortSignal.timeout(5000)` para validation do Turnstile.

Baixa 5.9 — No Input Sanitization em XML Upload

Problema

`diplomas/[id]/page.tsx` faz XML upload sem sanitization do conteúdo.

Impacto

XXE (XML External Entity) injection se content-type não for validado.

Diretriz de Solução

Validar Content-Type. Usar `fast-xml-parser` com `processEntities: false`.

Média 5.10 — Hardcoded Exempt Prefixes em CSRF

Problema

`csrf.ts` hardcoda prefixos isentos: `/api/portal/`, `/api/auth/`, `/api/ia/`.

Impacto

Novos endpoints em `/api/portal/` são automaticamente csrf-exempt.

Diretriz de Solução

Migrar para env var `NEXT_PUBLIC_CSRF_EXEMPT_ROUTES` com formato documentado.
