## Objetivo
Implementar trigger de imutabilidade pós-registro em diplomas + sistema de desbloqueio excepcional com janela temporária e auditoria completa.

## Entregáveis

### Migration SQL (`20260411_hard_lock_juridico.sql`)

- [x] Tabela `diploma_unlock_windows` — janelas temporárias de 5 min para edição
- [x] Trigger `trg_hard_lock_diploma` (BEFORE UPDATE) — bloqueia campos protegidos
- [x] RPC `desbloquear_diploma_para_edicao()` — abre janela + grava override + cadeia custódia
- [x] RPC `verificar_lock_diploma()` — consulta estado do lock
- [x] RLS + policies em `diploma_unlock_windows` e `validacao_overrides`

### Lógica do trigger
- **Status pós-registro:** `registrado`, `gerando_rvdd`, `rvdd_gerado`, `publicado`
- **Campos livres (sempre editáveis):** status, updated_at, pdf_url, qrcode_url, url_verificacao, data_publicacao, xml_url, codigo_validacao, codigo_validacao_historico
- **Todos os demais campos:** travados (curso_id, diplomado_id, datas, etc.)
- **Transição forward:** permitida (registrado→gerando_rvdd→rvdd_gerado→publicado)
- **Retrocesso de status:** bloqueado sem unlock window
- **Override humano:** justificativa ≥ 10 chars + audit trail completo
