## Entregáveis — Commit 613f151 (Sessão anterior, finalizado nesta)

### Novo: `src/lib/bry/timestamp-service.ts`
Três funções:
- `bryTimestampByContent()` — envia documento inteiro como multipart Blob (modo FILE)
- `bryTimestampByHash()` — envia apenas hash SHA256 hex string (modo HASH — eficiente)
- `aplicarCarimboDoTempo()` — conveniência: calcula SHA256 do XML, chama byHash, retorna `{ nonce, carimboBase64 }`

### Nova rota: `POST /api/diplomas/[id]/assinar/carimbo`
- Auth + CSRF + rate-limit (categoria "assinatura")
- Valida todos os passos do outbox como `finalizado`
- Idempotente: retorna sucesso se carimbo já existe
- Salva `carimbo_tempo_base64`, `carimbo_tempo_nonce`, `carimbo_tempo_at` em `xml_gerados`
- Registra cadeia de custódia: etapa `carimbo_do_tempo`

### DB (migration direta via Supabase MCP)
```sql
ALTER TABLE xml_gerados
  ADD COLUMN IF NOT EXISTS carimbo_tempo_base64 TEXT,
  ADD COLUMN IF NOT EXISTS carimbo_tempo_nonce TEXT,
  ADD COLUMN IF NOT EXISTS carimbo_tempo_at TIMESTAMPTZ;
```

### BRy Timestamp API
- Endpoint: `https://fw2.bry.com.br/api/carimbo-service/v1/timestamps`
- Auth: mesmo Bearer JWT do fluxo de assinatura (client_credentials)
- Retorno: `{ timeStamps: [{ nonce, content (base64) }] }`
- **Modo HASH usado** (mais eficiente — não transmite XML inteiro)

## Entregáveis — Commit df83975 (Principal desta sessão)

### Novo: `src/lib/bry/carimbo-pipeline.ts`
Dois utilitários internos sem contexto de Request/Auth:

**`aplicarCarimboXmlInterno(supabase, bryConfig, xmlGeradoId, xmlContentOverride?)`**
- Prioridade de conteúdo: override (do finalize) → arquivo_url (XML assinado storage) → conteudo_xml (fallback)
- Idempotente: retorna `{ ok: true, jaCarimbado: true }` se já existe
- Chamado diretamente pelo finalize com o XML assinado já em memória (sem re-fetch)

**`verificarEAvancarPacote(supabase, diplomaId)`**
- Verifica se TODOS os XMLs do diploma têm `status = 'assinado'` + `carimbo_tempo_base64`
- Se sim → atualiza diploma para `aguardando_envio_registradora`
- Guard: só avança se status atual for `assinado | aplicando_carimbo_tempo | em_assinatura`

### Modificação: `finalize/route.ts`
Após upload do XML assinado ao storage (quando `todosPassosFinalizados === true`):
1. Decodifica `xmlAssinadoBase64` → string UTF-8 (sem re-fetch)
2. Chama `aplicarCarimboXmlInterno()` **sincronamente** (não fire-and-forget)
3. Se OK → chama `verificarEAvancarPacote()`
4. Resposta agora inclui: `carimbo.aplicado`, `carimbo.erro`, `pacote_pronto`

**Decisão de design:** síncrono (não fire-and-forget) para garantir feedback imediato e evitar race conditions no Vercel serverless onde background tasks podem ser cortadas.

### Bug Fix: `carimbo/route.ts` — Prioridade de conteúdo corrigida
**Bug anterior:** usava `conteudo_xml` (XML ORIGINAL, não assinado) se presente → carimbo sobre XML sem assinatura!
**Fix:** inverte prioridade: `arquivo_url` (XML assinado no storage, pós-finalize) antes de `conteudo_xml`.

## Fluxo Completo Pós-Esta Sessão

```
Última assinatura (AD-RA eCNPJ) via BRy Signer
  ↓ BRy Finalize → XML assinado base64
  ↓ Upload para storage (xml-diplomas/assinado/{id}/)
  ↓ xml_gerados.status = 'assinado', arquivo_url = signed URL
  ↓ Auto-carimbo: SHA256(XML) → BRy Timestamp → carimbo_tempo_base64 salvo
  ↓ Se todos XMLs carimbados → diplomas.status = 'aguardando_envio_registradora'
  ↓ Frontend: seção "Prontos para Registradora"
  ↓ Operador clica "Gerar Pacote"
  ↓ ZIP montado: [XML assinado + .p7s carimbo + PDFs/A + manifest.json]
  ↓ URL download (1h) → operador baixa e envia à registradora
```

---

## Decisões Técnicas

| Decisão | Motivo |
|---------|--------|
| Carimbo síncrono no finalize (não fire-and-forget) | Vercel serverless corta background tasks após response; síncrono garante feedback e completude |
| xmlAssinadoUtf8 passado como override | Evita re-fetch do storage; XML já está em memória decodificado |
| Carimbo como .p7s separado no ZIP | Padrão CAdES; registradora pode validar timestamp independente do XML |
| arquivo_url priorizado sobre conteudo_xml | conteudo_xml = XML original sem assinatura; arquivo_url = XML assinado pós-finalize |
