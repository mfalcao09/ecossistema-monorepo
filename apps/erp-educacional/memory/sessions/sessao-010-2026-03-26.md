## Bug corrigido nesta sessão

### Erro de build — TypeScript type inference em validate-request.ts

**Problema:** O merge commit `19992c4` reintroduziu o erro de tipo em `src/lib/security/validate-request.ts:58`. O Zod `flatten().fieldErrors` retorna tipo genérico que o TS não consegue inferir como `string[]` quando o schema é `ZodSchema<T>`.

**Correção:** Type cast explícito `(msgs as string[] | undefined)` nas 3 funções:
- `validarBody` (linha 58)
- `validarQuery` (linha 124)
- `validarParams` (linha 190)

**Commit:** `08fb6c1` — "fix(ts): type cast fieldErrors em validate-request.ts (corrige build Vercel)"
