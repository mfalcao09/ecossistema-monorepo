## O que foi feito

### 6 ajustes aplicados ao FormularioRevisao e componentes relacionados

#### Ajuste 1 — Seção "Dados do Processo" simplificada
- Removidos: Turno, Período Letivo, Data de Colação
- Adicionados: Nome do Processo (readonly, auto-gerado: `{CPF} - {NOME COMPLETO}`) + Curso (readonly, do cadastro)
- Observações mantida

#### Ajuste 2 — Disciplinas: agrupamento + alfabético + editável
- Tabela flat substituída por grupos por período com header `Nº Período` ou label literal
- Dentro de cada grupo: ordem alfabética por nome
- Campos editáveis inline: nome, nota, docente, titulação (select com TITULACAO_DOCENTE_OPTIONS)
- Novo helper `mudaDisciplina(idx, campo, valor)` adicionado
- Novo enum `TITULACAO_DOCENTE_OPTIONS`: Doutor, Mestre, Especialista, Graduado, Residência

#### Ajuste 3 — Campos Acadêmicos Extras simplificados
- Removidos: Situação discente, CH integralizada, Data/Hora emissão histórico
- Mantido: Áreas do Curso (CRUD)
- Adicionado: Campos Personalizados (CRUD key-value genérico para campos que faltaram na extração)
- `campos_extras?: Array<{ chave: string; valor: string }>` adicionado ao DadosRevisao

#### Ajuste 4 — Renomeação
- "Estágio" → "Estágio Não-Obrigatório" (título da seção + botão "Adicionar")

#### Ajuste 5 — Assinantes somente leitura
- CRUD (add/edit/delete) removido completamente
- Exibe lista read-only dos assinantes configurados no sistema
- Nota apontando para "Configurações → Módulo Diploma"
- e-CNPJ da emissora também removido desta seção

#### Ajuste 6 — Checkboxes XML/Acervo movidos para o Dialog
- `DialogVisualizarDocumento`: novos props `destinoXmlInicial`, `destinoAcervoInicial`; novos estados `checkDestXml`, `checkDestAcervo`; `onConfirmar` agora passa `(confirmacao, destinoXml, destinoAcervo)`
- `CardArquivoClassificacao`: simplificado para somente-leitura — exibe nome, mime, tamanho e badges "XML" / "Acervo"
- `page.tsx`: `handleConfirmarComprobatorio` atualizado para receber destinos e chamar `setArquivosClassif`; dialog recebe valores iniciais do arquivo correspondente via `arquivo_index`; seção renomeada para "Arquivos processados"
