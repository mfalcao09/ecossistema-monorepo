## Etapa 1.2 — Rotas Next.js `/iniciar` + `/callback` (commit `f9739e46`)

### Decisões arquiteturais tomadas na sessão
- **1C — nonce de 1 uso embedado na query string do callback_url**. Railway trata URL como opaque, zero mudança Railway-side. Consumo atômico via `UPDATE ... WHERE callback_nonce_used_at IS NULL` no callback handler.
- **2B — signed URL TTL 600s (10 min)**. Suficiente pro Gemini baixar, curto pra minimizar exposição em caso de vazamento.
- **3B — Lock lógico via unique partial index**. Um índice para `processo_id IS NOT NULL`, outro para `processo_id IS NULL AND usuario_id IS NOT NULL` (caso drag-and-drop antes de criar o processo). Colisão → `23505` → `409` na rota.

### Entregas
1. **Migration** `20260408_sprint2_extracao_callback_nonce.sql` (aplicada via Supabase MCP + mirror no repo):
   - 6 colunas novas em `extracao_sessoes`: `callback_nonce`, `callback_nonce_used_at`, `iniciado_em`, `finalizado_em`, `processing_ms`, `erro_parcial`.
   - 3 unique partial indexes (lookup por nonce + 2 locks).
   - 1 CHECK constraint (`nonce_used_requires_nonce`).
   - `COMMENT ON` em todas as colunas novas.

2. **`src/lib/extracao/callback-auth.ts`** (110 linhas): helpers `gerarCallbackNonce()` (randomBytes 32), `comparaSecretoConstante()` (timingSafeEqual com length pre-check), `obterCallbackSecret()`, `montarCallbackUrl()`, `obterBaseUrlPublica()`.

3. **`POST /api/extracao/iniciar`** (~390 linhas):
   - `protegerRota` com `skipCSRF: true` (feedback memória).
   - Zod schemas com `.nullable().optional()`.
   - `gerarSignedUrls()` loopa arquivos e cria signed URLs TTL 600s.
   - `obterGeminiConfig()` lê `ia_configuracoes` joined com `ia_providers` (`modulo='diploma' AND funcionalidade='processamento_dados'`).
   - Handler de 8 passos: parse body → checa sessão existente → resolve config Gemini → gera nonce → INSERT sessão → gera signed URLs → `await fetch()` Railway → retorna 202.
   - **Helper `marcarSessaoComoErro`** (criado após review): só atualiza se ainda está em `'processando'` + warning log quando 0 rows (race benigna vencida pelo callback).

4. **`PUT /api/extracao/sessoes/[id]/callback`** (~194 linhas):
   - **Sem** `protegerRota` — autenticação é shared secret + nonce.
   - Valida header `x-extracao-callback-secret` constant-time.
   - Lê `nonce` da query string, exige mínimo 16 chars.
   - **UPDATE atômico** com `.eq('callback_nonce', nonce).is('callback_nonce_used_at', null)` → retorna 0 linhas se já consumido → 409.
   - Valida `sessao_id` do body vs URL, bumpa `version` para 2, marca `finalizado_em`.
   - Dois ramos: `status='erro'` (erro do Railway) vs `status='rascunho'` (sucesso com dados).

## Fluxo end-to-end resultante

```
UI → POST /api/extracao/iniciar
      ├─ INSERT extracao_sessoes (nonce, lock, processando)
      ├─ gera signed URLs TTL 600s dos arquivos
      └─ await fetch Railway /extrair-documentos (timeout 15s)
            └─ Railway 202 + processa em background
                  ├─ baixa signed_urls
                  ├─ Gemini 2.5 Flash extrai
                  ├─ agrega first-non-empty-wins
                  └─ PUT callback_url?nonce=XXX
                        └─ Next.js /callback
                              ├─ valida secret constant-time
                              ├─ consome nonce atomicamente
                              └─ UPDATE sessão → status rascunho, version=2
UI (polling) → GET /api/extracao/sessoes/[id]
```

## Decisões gravadas em memória
- **Nonce 1-uso como padrão pra callbacks internos** (Railway↔Next). Pode virar pattern pra BRy depois.
- **Fire-and-forget em Vercel serverless é anti-pattern.** Sempre `await` + `AbortSignal.timeout`. `maxDuration` não salva HTTP connections.
- **Unique partial indexes** como mecanismo de lock lógico supera tabela dedicada pra concorrência de baixa frequência.
- **Override humano universal** (princípio da sessão 022) reforçado: o callback é fonte da verdade, race vencida pelo callback é comportamento desejado.
