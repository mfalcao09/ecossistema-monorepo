- 4 índices + 3 column comments

**Módulo `src/lib/xml/validation/regras-negocio.ts` (novo, 93 linhas)**

Centraliza todas as regras de negócio do motor:
```typescript
export const REGRAS_NEGOCIO = {
  CARGA_HORARIA_INTEGRALIZADA_MENOR_QUE_TOTAL: 'CARGA_HORARIA_INTEGRALIZADA_MENOR_QUE_TOTAL',
} as const;

export class ValidacaoNegocioError extends Error {
  public readonly violacoes: ViolacaoRegra[];
  // ...
}

export function avaliarRegrasNegocio(
  dados: DadosDiploma,
  regrasIgnoradas: CodigoRegra[] = []
): ViolacaoRegra[]
```

Primeira regra: integralizada < total quando ambos > 0.

**`montador.ts`**

- Nova interface `MontarDadosDiplomaOptions` com `pular_regras_negocio?: CodigoRegra[]`
- Assinatura aceita 3º parâmetro `options`
- Ao final do pipeline, **dynamic `await import()`** evita ciclo circular module-level e dispara `avaliarRegrasNegocio`. Se há violações → `throw new ValidacaoNegocioError(violacoes)`.

**`POST /api/processos/[id]/gerar-xml/route.ts`**

- Lê `body.overrides: { codigo, justificativa }[]` (cada justificativa ≥ 10 chars)
- Passa `regrasIgnoradas` para `montarDadosDiploma`
- Catch `ValidacaoNegocioError` → retorna **422 estruturado**:
  ```json
  {
    "error": "Validação de regra de negócio",
    "tipo": "regra_negocio",
    "violacoes": [...],
    "mensagem_usuario": "Encontramos divergências..."
  }
  ```
- Após sucesso com overrides, faz `INSERT` em `validacao_overrides` por violação sobrescrita (auditoria completa: regra, valores originais, justificativa, usuario_id)

**Frontend `src/components/diploma/ModalOverrideRegra.tsx` (novo, 117 linhas)**

- Modal controlado com state `{[codigo]: justificativa}` por violação
- Header com `ShieldAlert`, cards individuais com `AlertTriangle`, valores originais em `<pre>` JSON
- Validação `todasJustificadas = violacoes.every(v => trim().length >= 10)` habilita o botão Confirmar
- Exporta `ViolacaoRegraResposta` e `OverridePayload`

**Integração em `diploma/diplomas/[id]/page.tsx`**

- Refatorou `gerarXml` em `chamarGerarXml(overrides)` que detecta 422 + `tipo: "regra_negocio"` e abre o modal
- `confirmarOverride` re-chama a API com os overrides; `cancelarOverride` apenas fecha
- Modal renderizado no topo do `PainelAcoes`

### Bug #G — Atos regulatórios do curso → tabela dedicada

**Migration `20260407_atos_curso.sql`** (aplicada via Supabase MCP)

Tabela espelhando `credenciamentos`:
- FK `curso_id` ON DELETE CASCADE
- `tipo` CHECK ('Autorizacao' | 'Reconhecimento' | 'RenovacaoReconhecimento')
- Demais colunas: `tipo_ato`, `numero`, `data`, `veiculo_publicacao`, `numero_dou`, `data_publicacao_dou`, `secao_dou`, `pagina_dou`

**Backfill idempotente** (3 INSERT...SELECT...WHERE NOT EXISTS) a partir dos campos planos da `cursos`. Resultado verificado via Supabase MCP:
- Autorizacao: 8 registros
- Reconhecimento: 8 registros
- RenovacaoReconhecimento: 6 registros

**`montador.ts`**

- Interface `AtosCurso { autorizacao?, reconhecimento?, renovacao_reconhecimento? }`
- Helper `linhaAtoCursoParaAto()` (espelho de `linhaCredenciamentoParaAto`)
- Função async `buscarAtosCurso(supabase, cursoId)` — lê da tabela ordenado por `data ASC` e aplica:
  - Autorização: primeira (mais antiga)
  - Reconhecimento: primeiro (mais antigo)
  - Renovação: última (mais recente) — opcional
- Chamada `const atosCurso = await buscarAtosCurso(supabase, cur.id)` adicionada ao fluxo principal logo após `cur = diplomaData.cursos`
- Bloco `curso.autorizacao/reconhecimento/renovacao_reconhecimento` no payload usa padrão `atosCurso.* ?? fallback_planos` para compatibilidade com cursos legados

## Decisões arquiteturais importantes desta sessão

1. **Override humano universal:** é princípio fundamental do ERP, não só do motor XML. A tabela `validacao_overrides` é genérica (`entidade_tipo`/`entidade_id`/`regra_codigo`) e deve ser reutilizada por qualquer regra de negócio futura — financeiro, acadêmico, regulatório.
2. **Justificativa mínima 10 chars:** CHECK constraint no banco — não pode ser desabilitado pelo frontend.
3. **Dynamic import dentro da função:** padrão para módulos de validação que dependem do mesmo arquivo que os instancia (evita ciclo).
4. **Padrão `tabela_dedicada ?? fallback_planos`:** mantém migração gradual sem big-bang. Os cursos legados continuam funcionando até serem migrados explicitamente.
