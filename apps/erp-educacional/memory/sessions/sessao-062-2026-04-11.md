## Contexto
Bug recorrente: modal "Criar processo com pendências" listava 4/4 comprobatórios faltantes, enquanto sidebar mostrava "4/4 confirmados" com checkmarks verdes. Havia sido parcialmente corrigido nas sessões 047 e 049b, mas reapareceu.

## Diagnóstico — Causa Raiz
O fluxo de extração (drag-and-drop → Railway → revisão) **nunca criava linhas em `processo_arquivos`**. O único INSERT existia na rota legada `/api/processos/[id]/arquivos`. 

Consequência em cadeia:
1. `processo_arquivos` vazio para a sessão → `arquivosClassif = []` no React
2. memo `arquivosComConfirmacoes` sem nada para enriquecer → `tipo_xsd` não propagado
3. Gate validator lê `processo_arquivos.tipo_xsd` do banco → vê 0 docs → bloqueia

Sidebar mostrava "4/4 confirmados" porque lia de `dados_extraidos.comprobatorios_detectados` (JSONB in-memory) — caminho completamente separado.

## Entregas

### Fix: INSERT processo_arquivos no POST /api/extracao/iniciar
- Adicionado bloco INSERT (step 5b) em `src/app/api/extracao/iniciar/route.ts`
- Cria uma linha por arquivo logo após criação da sessão
- Non-blocking: falha é logada mas não impede extração
- 17 linhas populadas via SQL direto para a sessão ativa (`39c0718e`)

### Verificações
- `npx tsc --noEmit` — limpo (erros são apenas vitest module declarations pré-existentes)
- Deploy Vercel READY em 54s

## Decisões Técnicas
1. INSERT é non-blocking por design — se falhar, gate pode ser ultrapassado com override humano
2. Campos `tipo_xsd`, `destino_xml`, `destino_acervo` ficam NULL no INSERT inicial; são preenchidos pelo PUT da revisão (auto-save)
3. `processo_id` pode ser NULL neste ponto — será preenchido pela RPC `converter_sessao_em_processo`

## Por que fixes anteriores não resolveram
- Sessão 047: corrigiu path `diplomado.nome_completo → aluno.nome` e filtro `destino_xml` — mas sem linhas no banco, não havia o que filtrar
- Sessão 049b: corrigiu sync tipo_xsd nas confirmações — mas sem linhas no banco, não havia onde gravar
