## O que foi feito nesta sessão

### 1. Reescrita completa webhook.ts (commit e3578d1)

Substituição total de todas as referências ao schema antigo:

| Schema antigo (`chat_*`) | Schema correto (`atendimento_*`) |
|--------------------------|----------------------------------|
| `chat_channels` | `atendimento_inboxes` |
| `chat_contacts.phone` | `atendimento_contacts.phone_number` |
| `chat_conversations.channel_id` | `atendimento_conversations.inbox_id` |
| `chat_messages.wamid` | `atendimento_messages.channel_message_id` |
| `sender_type: 'customer'` | `sender_type: 'contact'` |
| `tenant_id` | removido (schema single-tenant) |
| lookup por `meta_phone_number_id` | lookup por JSONB `provider_config->>phone_number_id` |

Inbox lookup com PostgREST JSONB filter:
```typescript
await (supabase as any)
  .from('atendimento_inboxes')
  .select('id')
  .filter('provider_config->>phone_number_id', 'eq', phoneNumberId)
  .eq('enabled', true)
  .maybeSingle()
```

### 2. Fallback de inbox (commit 7df5790)

Meta dispatcher de teste usa `phone_number_id: "123456123"` (placeholder falso).
Em produção, usa o ID real `938274582707248`.

Fallback adicionado: se não achar pelo `phone_number_id` exato, usa o primeiro
inbox WhatsApp ativo do banco.

```typescript
const { data: inboxFallback } = !inbox
  ? await (supabase as any)
      .from('atendimento_inboxes')
      .select('id')
      .eq('channel_type', 'whatsapp')
      .eq('enabled', true)
      .limit(1)
      .maybeSingle()
  : { data: null }
const inboxResolvido = inbox ?? inboxFallback
```
