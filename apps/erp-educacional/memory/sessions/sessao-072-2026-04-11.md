# Sessão 072 — 2026-04-11

## Objetivo
Fix Bug #3 (continuação da sessão 071): botões de correção da auditoria XSD abriam o formulário legado vazio ao invés do formulário de revisão da extração com os dados da IA.

## Sprint/Epic
- **Sprint 2 / Epic 2.1** — Motor XML polish (fixes da pipeline de auditoria)

## Problema resolvido
Após o fix da sessão 071, os botões "Editar dados pessoais", "Editar histórico / disciplinas" e "Preencher docentes" abriam `/diploma/processos/{processoId}` — o formulário legado de entrada manual. Para diplomas criados pelo novo fluxo de extração, esse formulário está completamente vazio porque os dados ficam em `extracao_sessoes.dados_confirmados`, não nas tabelas antigas do processo.

**Evidência:** Screenshot de Marcelo mostrando `/diploma/processos/{id}` com "0 disc. | CH 0h | 0 verificados | 0 pendentes".

**Mensagem Marcelo:** _"mas meu anjo, a tela que você me direcionou pertence ao fluxo antigo. Ela sequer tem as informações do novo fluxo de extração. Nós precisamos acessar o novo formulário de extração, para editar os dados lá!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!"_

**Causa raiz:** O destino correto é `/diploma/processos/novo/revisao/{sessaoId}` (FormularioRevisao com todos os dados extraídos pela IA). O `sessaoId` já estava disponível como `extracao?.id` no estado da página, mas não estava sendo passado ao `PainelAuditoria`.

## Solução
Adição de `sessaoId` à cadeia de props já refatorada na sessão 071:

1. **`BotaoCorrecao`**: Roteamento dual:
   - `sessaoId` presente → `/diploma/processos/novo/revisao/{sessaoId}` (novo fluxo)
   - `sessaoId` ausente → `/diploma/processos/{processoId}` (fallback legado)
2. **`IssueRow`**: `sessaoId` adicionado ao tipo
3. **`GrupoExpandivel`**: `sessaoId` adicionado ao tipo e repassa
4. **`PainelAuditoriaProps`**: `sessaoId?: string | null` adicionado
5. **`PainelAcoes` (page.tsx)**: `sessaoId?: string | null` adicionado, passa a `PainelAuditoria`
6. **Componente principal (page.tsx)**: Passa `sessaoId={extracao?.id}` ao `PainelAcoes`

### Fonte do sessaoId
```ts
// page.tsx — já existia desde sessão 038+
const [extracao, setExtracao] = useState<ExtracaoSessao | null>(null)
// ...
<PainelAcoes sessaoId={extracao?.id} ... />
```
