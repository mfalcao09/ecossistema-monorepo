## O Bug

**Sintoma:** Badge "Extensão não detectada" mesmo com extensão instalada e ativa no Chrome.

**Causa raiz:** O código de detecção original executava apenas uma tentativa, com delay de 500ms:
```typescript
const t = setTimeout(check, 500);
```
A extensão BRy Signer injeta `window.BryWebExtension` de forma assíncrona após o carregamento da página. Dependendo do estado do navegador (cold start, cold extension context), 500ms pode ser insuficiente. Uma única tentativa sem retry deixa a janela de injeção passar silenciosamente.

---

## Fix Aplicado

**Arquivo:** `src/app/(erp)/diploma/assinaturas/page.tsx`

### 1. Estado `detectandoExtensao`
Adicionado para controlar loading state:
```typescript
const [detectandoExtensao, setDetectandoExtensao] = useState(true);
```

### 2. Função `verificarExtensao` (retry loop)
Substituído o `useEffect` com 1 timeout por uma função `useCallback` com loop de retry:
- **16 tentativas** × 500ms = **até 8 segundos** de tolerância
- Loop `for` assíncrono com `await new Promise(resolve => setTimeout(resolve, 500))`
- Ao detectar `window.BryWebExtension.isExtensionInstalled() === true`, para imediatamente e carrega certificados
- Se esgotar as 16 tentativas, seta `extensaoInstalada = false` e `detectandoExtensao = false`

### 3. Badge "Verificando…"
```tsx
{detectandoExtensao ? (
  <span className="animate-pulse">Verificando…</span>
) : extensaoInstalada ? (
  <span>Extensão ativa</span>
) : (
  <span>Extensão não detectada</span>
)}
```

### 4. Botão "Verificar novamente"
Exibido no bloco de aviso (apenas quando `!detectandoExtensao && !extensaoInstalada`):
```tsx
<button onClick={verificarExtensao}>Verificar novamente</button>
```
Permite re-detectar sem recarregar a página.

### 5. Mensagem atualizada
Texto ajustado para mencionar "habilitada para este domínio" — orientação mais precisa para o usuário verificar as permissões da extensão.
