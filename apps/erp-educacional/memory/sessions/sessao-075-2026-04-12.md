## Bugs Diagnosticados e Corrigidos

### Bug #1 — Navegação lista processos
**Sintoma:** Após "Confirmar dados" na revisão, redireciona para lista. Clicar no processo abre pipeline em vez da revisão.
**Causa:** Condição de navegação exigia `p.status === 'em_extracao' && p.sessao_id`.
**Fix:** `p.sessao_id ? /revisao/${p.sessao_id} : ...` (sessao_id presente → sempre revisão, independente de status).
**Arquivo:** `src/app/(erp)/diploma/processos/page.tsx`

### Bug #2 — Links "Ver chat IA" e "Chat IA de extração" no pipeline
**Sintoma:** 3 links no pipeline ([id]/page.tsx) apontavam para `/diploma/processos/${proc.id}` (formulário antigo).
**Fix:** `extracao?.id ? /diploma/processos/novo/revisao/${extracao.id} : /diploma/processos/${proc.id}`.
**Arquivo:** `src/app/(erp)/diploma/diplomas/[id]/page.tsx`

### Bug #3 — Redirect pós-confirmar revisão
**Sintoma:** Após clicar "Confirmar dados" com sucesso, o router.push ia para `/diploma/processos` (lista).
**Fix:** Extrair `diploma_id` do response e navegar para `/diploma/diplomas/${diplomaId}` (pipeline).
**Arquivo:** `src/app/(erp)/diploma/processos/novo/revisao/[sessaoId]/page.tsx`

### Bug #4 (a) — Auditoria: "Nenhum genitor informado"
**Sintoma:** Auditoria reportava 0 genitores apesar de JAIR (M) e ELIZETE (F) confirmados.
**Causa raiz:** `filiacoes` era canônico (2 rows) mas `diplomados.nome_pai/mae` estavam null (legado). A rota de auditoria lia só `diplomadoExtras` sem consultar `filiacoes`.
**Fix:** Adicionado query `filiacoes` no Promise.all da rota de auditoria; nome_pai/mae usa filiacoes como fonte primária com fallback para legado.
**Arquivo:** `src/app/api/diplomas/[id]/auditoria/route.ts`
**SQL:** Dados OK (banco já tinha os 2 genitores).

### Bug #4 (b) — Auditoria: Histórico com 4 erros críticos falsos
**Sintomas:** código_curriculo crítico, data_expedição crítico, carga_horária 0 (form mostrava 4560h), 55 disciplinas sem docente.
**Causas raiz (3 subcausas):**
1. `codigo_curriculo` e `data_expedicao` estavam como `critico` mas deveriam ser avisos (disponíveis apenas no momento da assinatura).
2. `carga_horaria_integralizada` null no diploma — RPC lia chave errada em `dados_confirmados`.
3. 55 disciplinas com `docente_nome = null` — RPC lia chave `nome_docente` mas o extrator v3/reducer salvava `docente`.

**Fixes:**
- `src/lib/auditoria/grupos/historico.ts`: `codigo_curriculo` e `data_expedicao` → severidade `aviso`.
- `src/app/api/diplomas/[id]/auditoria/route.ts`: fallback `chFromDisciplinas = sum(carga_horaria_aula)`.
- SQL migration `fix_rpc_comprobatorios_cast_s075`: COALESCE `docente` / `nome_docente`; CH lido de top-level e de `v_academicos`; `tipo_xsd::tipo_documento_comprobatorio` cast.
- SQL data fix `fix_data_kauana_s075`: 55 `diploma_disciplinas.docente_nome` atualizados; `diplomas.carga_horaria_integralizada = 4560`.

### Bug #4 (c) — Auditoria: 0 comprobatórios
**Sintoma:** Auditoria reportava "Nenhum documento obrigatório" apesar de 4 docs confirmados no gate.
**Causa raiz:** RPC `converter_sessao_em_processo` nunca fazia INSERT em `diploma_documentos_comprobatorios` (só em `processo_arquivos`).
**Fix:** SQL migration `fix_rpc_comprobatorios_cast_s075` corrigiu o INSERT com cast correto. SQL data fix inseriu os 4 comprobatórios da sessão da Kauana.

## Aprendizados Técnicos

- **JSONB key mismatch**: extrator v3 salva `docente`; RPC antiga lia `nome_docente` → 55 disciplinas sem docente. COALESCE ambos.
- **Enum cast obrigatório em PL/pgSQL**: `processo_arquivos.tipo_xsd` é `text`; `diploma_documentos_comprobatorios.tipo_xsd` é enum → `::tipo_documento_comprobatorio` necessário mesmo em SELECT para INSERT.
- **PL/pgSQL late compilation**: funções com erro de tipo são aceitas no CREATE mas falham no runtime — testar sempre com dados reais.
- **FUSE delete-deny**: `.git/index.lock` não pode ser removido no bind mount → workaround /tmp clone canônico.
- **Repo name**: `mfalcao09/diploma-digital` (NÃO `erp-educacional`).
