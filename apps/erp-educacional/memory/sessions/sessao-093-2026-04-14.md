## 3. Fixes na `bootstrap_session()`

**Problema:** `match_ecosystem_memory_keyword` usava `plainto_tsquery` (AND entre todos os termos). Busca por "diploma digital XML geração assinatura BRy" retornava apenas 1 resultado.

**Fix (migration `fix_bootstrap_session_v2`):**
- Substituiu `plainto_tsquery` por `websearch_to_tsquery` (OR implícito entre termos não-quotados)
- Adicionou ILIKE em títulos para siglas (BRy, XML, XSD, FIC) não tratadas pelo stemmer português
- Reescreveu `bootstrap_session()` com estrutura CTE (structural UNION contextual) — evita syntax error PostgreSQL com UNION+LIMIT em plpgsql

---

## 4. Decisões desta Sessão

| Decisão | Raciocínio |
|---------|-----------|
| `verify_jwt = false` na Edge Function | Chamada vem do trigger DB, sem JWT de usuário disponível |
| Segredo `x-internal-secret` hardcoded no trigger | Infraestrutura interna, não exposta publicamente; aceitável |
| `outputDimensionality: 768` explícito no Gemini | Garante compatibilidade com coluna `vector(768)` mesmo se default mudar |
| Idempotência por `embedding IS NOT NULL` | Evita re-embedar memórias já processadas (on-insert e on-call) |
| `SECURITY DEFINER` na função trigger | Garante acesso ao schema `net` mesmo para roles restritas |
