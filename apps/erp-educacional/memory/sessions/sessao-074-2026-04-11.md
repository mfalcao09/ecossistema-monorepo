## Contexto e Motivação

Kauana tinha 16 documentos upados e a sessão foi "perdida" — não havia como rastrear o processo que a extração gerou, pois `processos_emissao` só era criado no final (ao confirmar dados). Se a sessão ficasse bloqueada, o processo inteiro sumia. Decisão: **processo nasce no momento do upload**, em status `em_extracao`, e fica rastreável desde o início.

Marcelo aprovou todos os 5 épicos em um único sprint imediato.

## Épicos Entregues

### Épico A: DB Migration (aplicado diretamente no Supabase)
- `processos_emissao.nome` DROP NOT NULL (nome só existe após confirmação)
- `processos_emissao.sessao_id uuid REFERENCES extracao_sessoes(id) ON DELETE SET NULL`
- `UNIQUE INDEX uq_processos_emissao_sessao_id WHERE sessao_id IS NOT NULL AND status != 'cancelado'`

### Épico B: POST /api/extracao/iniciar + RPC update
**Arquivo:** `src/app/api/extracao/iniciar/route.ts`
- Passo 5b (novo): INSERT em `processos_emissao` com `status='em_extracao'` imediatamente após criar `extracao_sessoes`
- UPDATE `extracao_sessoes.processo_id` com o ID recém-criado
- `processoIdEfetivo` propagado para `processo_arquivos` (era `null` antes)

**RPC `converter_sessao_em_processo`** (Supabase):
- Agora: IF `v_sessao.processo_id IS NOT NULL` → UPDATE (preenche nome, curso, status='rascunho')
- ELSE → INSERT (backward-compatible para sessões legadas sem processo)
- Elimina duplicação: sessão com processo_id já existente nunca gera processo novo

### Épico C: Tela de Revisão (`page.tsx`)
**Arquivo:** `src/app/(erp)/diploma/processos/novo/revisao/[sessaoId]/page.tsx`
- Botão CTA condicional: `sessao.processo_id` presente → "Confirmar dados" | ausente → "Criar processo"
- Loading label condicional: "Confirmando dados..." | "Criando processo..."
- Banner `convertido_em_processo`: título alterado de "Conversão concluída" para "Revisão pós-auditoria"
- Texto do banner: removida frase "será resolvida na próxima sprint"

### Épico D: RLS + Cleanup 90 dias
- Status `em_extracao` já em CHECK constraint (pré-existia) — sem migration
- Cleanup 90 dias: tech-debt no radar (pg_cron ou Edge Function), não implementado nesta sessão

### Épico E: Lista de Processos — navegação e exibição
**Arquivo:** `src/app/api/processos/route.ts`
- `sessao_id` adicionado ao SELECT e ao `formatted` map (`p.sessao_id ?? undefined`)
- Interface `ProcessoResponse`: `nome: string | null`, `sessao_id?: string`

**Arquivo:** `src/app/(erp)/diploma/processos/page.tsx`
- Interface `Processo`: `nome: string | null`, `sessao_id?: string`
- Null nome fallback: `p.nome ?? (p.status === "em_extracao" ? "Extraindo documentos…" : "—")` com classe italic violet
- Click navigation: status `em_extracao` + `sessao_id` → `/diploma/processos/novo/revisao/${sessao_id}`
- ArrowRight button: mesma lógica de navegação + title "Continuar revisão"
