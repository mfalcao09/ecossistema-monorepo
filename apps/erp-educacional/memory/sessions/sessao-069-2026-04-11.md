## Objetivo
Fix C: Quando um processo foi criado via novo fluxo de extração IA, a página `/diploma/processos/[id]` mostrava "Confirmar e Criar Processo" (errado). Não havia caminho para o botão "Gerar XML". Corrigir navegação para ir direto à pipeline.

## Problema Identificado
O fluxo novo (drag-drop → extração → revisão → `converter_sessao_em_processo`) cria automaticamente tanto o `processos_emissao` quanto o `diplomas` linkado. Mas a página de listagem e o formulário antigo não sabiam que o diploma já existia, então:
- List page (`/processos`) abria o formulário antigo ao clicar no processo
- Formulário antigo mostrava "Confirmar e Criar Processo" (criaria duplicata)
- Usuário ficava sem acesso ao botão Gerar XML da pipeline

## Solução — 4 arquivos alterados

### 1. `src/app/api/processos/[id]/route.ts`
GET do processo individual agora retorna `diploma_id` (primeiro diploma vinculado ao processo).

### 2. `src/app/api/processos/route.ts`
List API agora inclui join FK `diplomas!diplomas_processo_id_fkey(id)` e retorna `diploma_id` por processo.

### 3. `src/app/(erp)/diploma/processos/[id]/page.tsx`
Formulário antigo: quando `processo.diploma_id` existe, mostra botão **"Ir para Pipeline (Gerar XML / Assinar)"** em vez de "Confirmar e Criar Processo".

### 4. `src/app/(erp)/diploma/processos/page.tsx`
List page: `onClick` navega direto para `/diploma/diplomas/{diploma_id}` quando o processo tem diploma linkado.
