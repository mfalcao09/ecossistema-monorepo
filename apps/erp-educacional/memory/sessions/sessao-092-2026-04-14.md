## Design Aprovado — Modelo de Cobrança FIC

### Fluxo de vida de uma mensalidade:
```
Dia 01: Bolepix emitido (vencimento dia 08, desconto pontualidade %)
Dia 08: Último dia com desconto — Inter aplica automaticamente (quantidadeDias=0)
Dia 09 00:01: Boleto expirado (numDiasAgenda=0) → sistema detecta inadimplência
Dia 09 08:00: cron-inadimplencia → marca 'vencido', calcula encargos, e-mail dia 1
Diário 08:00: cron-regua → verifica dias de atraso, dispara mensagem se for dia de régua
Sob demanda: gerar-pix-demanda → PIX válido até 23:59 do dia solicitado (1 por dia)
Dia 31: restricao_aluno tipo=bloquear_portal aplicada automaticamente
Dia 00:01: cron-expirar-pix → marca PIXs de ontem como 'expirado'
```

### Parâmetros financeiros:
- Multa: **10% fixo**, aplicada 1x no 1º dia de atraso
- Juros: **2% ao mês** = 2/30% por dia, acumulado linearmente
- Desconto pontualidade: **por aluno**, definido na rematrícula (0-25%)
- Boleto: **NAOTEMMULTA + ISENTO** (encargos só no PIX sob demanda)
- PIX sob demanda: **1 por cobrança por dia** (UNIQUE), expira 23:59

### Régua de cobrança aprovada:
| Dia de atraso | Canal | Tom |
|---|---|---|
| 1 (dia 09) | E-mail | Amigável — "Quer PIX para hoje?" |
| 3 (dia 11) | E-mail | Lembrete |
| 7 (dia 15) | E-mail | Atenção + saldo |
| 12 (dia 20) | E-mail | Urgência — cita rematrícula |
| 17 (dia 25) | E-mail | Institucional — letterhead |
| 22 (dia 30) | E-mail | Pré-restrição — avisa bloqueio |
| 23 (dia 31) | Sistema | Aplica restrição de portal |
| Fase B | WhatsApp | Meta Business API (não implementado ainda) |

### Restrições legais confirmadas por Marcelo:
- **NÃO pode bloquear matrícula já realizada** (pacificado na jurisprudência)
- **PODE impedir rematrícula** semestre seguinte
- **PODE bloquear portal do aluno** (acesso a documentos)
- Futuramente: **bloquear catraca** (quando catracas forem instaladas)

### 2. `api/financeiro/emit-boletos.py` — AJUSTADO
Mudanças aplicadas:
- `multa: NAOTEMMULTA` (era PERCENTUAL 2%)
- `mora: ISENTO` (era TAXAMENSAL 1%)
- `numDiasAgenda: 0` (era 60 — boleto agora expira no dia do vencimento)
- Bloco `desconto` Inter adicionado: `PERCENTUAL, quantidadeDias=0, taxa=X%`
- SELECT em `alunos` inclui `desconto_pontualidade`
- `_registrar_cobranca` inclui `tipo="mensalidade"` e `desconto_aplicado`
- Bug fix: `mensagem` → `conteudo` no insert de `comunicacoes`
- Bug fix: `tipo` obrigatório adicionado ao insert de `comunicacoes`
- E-mail atualizado para mostrar desconto aplicado e aviso de expiração

### 3. `api/financeiro/cron-inadimplencia.py` — NOVO
- Roda: todo dia 08:00 (via Vercel Cron)
- Busca cobranças enviadas/geradas com `data_vencimento < hoje`
- Marca status como `'vencido'`
- Calcula encargos: multa 10% + juros 2%/mês pro rata
- INSERT em `inadimplencia_diaria` (upsert idempotente)
- Se `dias_atraso == 1`: envia e-mail amigável via Resend
- Registra em `comunicacoes` com `tipo='cobranca_inadimplencia'`

### 4. `api/financeiro/gerar-pix-demanda.py` — NOVO
- Endpoint: `POST /api/financeiro/gerar-pix-demanda`
- Body: `{ aluno_id, canal: 'portal'|'whatsapp', cobranca_id? }`
- Idempotente: se já existe PIX ativo para hoje → retorna o existente
- Calcula valor exato do dia (principal + multa + juros acumulados)
- Emite cobrança Inter: `numDiasAgenda=0`, vencimento=hoje, NAOTEMMULTA+ISENTO
- UNIQUE `(cobranca_id, data_validade)` — garante 1 PIX/dia/cobrança
- Retorna: `{ pix_copia_cola, valor, valido_ate, dias_atraso, ja_existia }`
- `your_number` formato: `PD{ALUNO5}{YYYYMMDD}` = 15 chars (limite Inter)

### 5. `api/financeiro/cron-regua.py` — NOVO
- Roda: todo dia 08:00 (via Vercel Cron)
- Para cada cobrança com `status='vencido'`:
  - Calcula `dias_atraso = hoje - data_vencimento`
  - Verifica se é dia de disparo da régua (1/3/7/12/17/22)
  - Verifica se já enviou hoje (idempotência via tabela `comunicacoes`)
  - Envia e-mail com tom específico por dia
  - Dia 23: aplica `restricoes_aluno tipo='bloquear_portal'`
- 6 tons de e-mail: amigavel/lembrete/atencao/urgencia/institucional/pre_restricao
- WhatsApp: placeholder com log (Fase B — Meta Business API)

### 6. `api/financeiro/cron-expirar-pix.py` — NOVO
- Roda: todo dia às **00:01** (via Vercel Cron `1 0 * * *`)
- Busca `pix_demanda WHERE status='ativo' AND data_validade < hoje`
- Marca como `'expirado'`
- Garante que PIXs de ontem não apareçam como válidos no portal
- O Inter já baixa automaticamente (numDiasAgenda=0), este cron espelha no banco

### 7. `vercel.json` — ATUALIZADO
Novos crons registrados:

{ "path": "/api/financeiro/cron-regua",         "schedule": "0 8 * * *"  },

## Decisões Técnicas Desta Sessão

1. **Supabase ERP correto**: `ifdnjieklngcfodmtied` (diploma-digital, sa-east-1)
   — NÃO `bvryaopfjiyxjgsuhjsb` (Intentus) — verificado e corrigido nesta sessão

2. **quantidadeDias=0**: Desconto Inter válido até o próprio vencimento inclusive
   — Soluciona o problema de o Inter exigir `limitDate < vencimento`

3. **PIX your_number formato**: `PD{ALUNO5}{YYYYMMDD}` = 15 chars (limite exato do Inter)
   — Diferente do boleto: `FIC{ALUNO8}{YYYYMM}` = 15 chars

4. **Idempotência dupla**:
   - `inadimplencia_diaria`: UNIQUE (cobranca_id, data_referencia) → upsert
   - `pix_demanda`: UNIQUE (cobranca_id, data_validade) → verifica antes de criar
   - `cron-regua`: verifica `comunicacoes` antes de disparar (evita duplo envio)
   - `restricoes_aluno`: verifica existente antes de inserir

5. **Fase A/B separadas**: WhatsApp (Fase B) tem placeholder com log — não bloqueia deploy
