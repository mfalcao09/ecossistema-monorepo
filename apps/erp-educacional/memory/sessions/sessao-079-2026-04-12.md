## Investigação — Causa Raiz Real

### Descoberta 1: Script Bridge Externo
O `window.BryWebExtension` **NÃO é injetado pela extensão Chrome** diretamente. É criado por um script JS externo hospedado em:
```
https://www.bry.com.br/downloads/extension/v2/api/v1/extension-api.js
```
Sem carregar esse script, `window.BryWebExtension` é **sempre undefined**.

**Evidência:** No exemplo oficial BRy (`geracao-diploma-extensao-master.zip`), o `public/index.html` carrega o script na linha 8:
```html
<script src="https://www.bry.com.br/downloads/extension/v2/api/v1/extension-api.js"></script>
```

### Descoberta 2: API é Promise-based
`isExtensionInstalled()` retorna `Promise<boolean>`, não `boolean`. O código anterior tratava como síncrono.

### Descoberta 3: Fluxo Correto
1. Carregar `extension-api.js` (cria `window.BryWebExtension`)
2. `await isExtensionInstalled()` → boolean
3. `await installComponents()` → garante módulo nativo
4. `await listCertificates()` → lista certificados do Token A3
