## O que foi entregue

### Bug 1 — Cadastro de usuários não salvava (CSRF 403)
- **Causa raiz:** `page.tsx` de configurações/usuários usava `fetch()` nativo nas chamadas POST e PATCH.
- Ambas as rotas da API têm `validarCSRF(request)` — exigem token CSRF no header.
- `fetch()` nativo não envia o token; apenas `fetchSeguro()` o lê do cookie e injeta automaticamente.
- **Fix:** Substituído `fetch()` por `fetchSeguro()` nas duas mutações (criar + editar).
- **Fix secundário:** Leitura do erro corrigida de `d.error` para `d.error ?? d.erro` (API usa `erro` no Supabase e `error` no Zod).

### Bug 2 — Schema Zod com roles desatualizadas ("Dados Inválidos")
- **Causa raiz:** `criarUsuarioSchema` aceitava apenas `['admin', 'secretario', 'diretor', 'visualizador']` — roles antigas.
- O frontend enviava valores como `admin_instituicao`, `aux_secretaria`, `diretoria`, etc. — todos rejeitados pelo Zod.
- **Fix:** Enum atualizado para os 10 perfis reais: `admin_instituicao`, `aux_bibliotecaria`, `aux_financeiro`, `aux_secretaria`, `bibliotecaria`, `cadastramento`, `comunidade`, `coordenacao_curso`, `diretoria`, `estudantes`.

### Feature — Indicador de requisitos de senha + confirmação
- Checklist visual em tempo real (aparece ao digitar): 5 requisitos com ✓/✗ coloridos.
- Campo "Confirmar Senha" com borda verde/vermelha e mensagem instantânea.
- Botão "Cadastrar usuário" desabilitado até todos os requisitos atenderem E senhas coincidirem.
- Schema Zod atualizado para incluir símbolo (`/[^A-Za-z0-9]/`) como requisito obrigatório.
- Frontend e servidor agora totalmente alinhados nos requisitos de senha.

### Fix — Título do site
