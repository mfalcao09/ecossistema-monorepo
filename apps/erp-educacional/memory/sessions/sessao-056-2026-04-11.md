## Entregas

### Epic 1.2 — Supabase Vault
- **Migration SQL:** `get_vault_secret()` helper + RPCs vault-aware (encrypt_pii, decrypt_pii, hash_cpf)
- **Vault-first pattern:** RPCs leem chave do Vault primeiro, fallback para parâmetro (DEFAULT NULL)
- **pii-encryption.ts V2:** `obterChaveFallback()` retorna undefined se env var ausente — Vault fornece
- **Rota temporária** `POST /api/admin/migrate-secrets-to-vault` (runtime, não expõe chave)
- **RPC temporária** `insert_vault_secret` (wrapper vault.create_secret)
- **Migração executada:** PII_ENCRYPTION_KEY inserida no Vault (UUID 0bf1a901-3a0d-...)
- **Roundtrip verificado:** encrypt→decrypt via Vault funciona sem passar chave
- **Limpeza:** rota + RPC temporárias removidas após uso

### Epic 1.3 — Railway Security (Audit Trail + Rate Limiting)
- **Audit trail:** tabela `extracao_sessoes_audit` + trigger AFTER UPDATE automático
- **RPC `update_extracao_with_audit`:** injeta contexto de auditoria + UPDATE em transação única
- **supabase-writer.js V2:** usa RPC auditada (não mais UPDATE direto)
- **Rate limiter:** sliding window in-memory (30/60/120 req/min), middleware Express
- **Audit context middleware:** captura IP, gera requestId, chama setAuditContext()

## Decisões Técnicas
1. **PostgREST session isolation:** Cada chamada JS cria nova sessão DB — resolvido wrapping em RPC única
2. **Vercel Sensitive vars:** write-only by design — solução: rota serverless que lê em runtime e grava no Vault
3. **skipCSRF não é export válido** do Next.js 15 Route — CSRF bypass vem do middleware, não da rota

## Problemas Encontrados
- Build Vercel falhou por `export const skipCSRF = true` (export inválido) — removido, fix em 626a80a
- Verificação da rota de migração reportou "chave incorreta" — falso positivo: PostgREST retorna bytea como hex, RPC espera text. Fluxo real funciona perfeitamente.
