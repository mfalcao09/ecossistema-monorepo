## Padrões Cross-Project (aplicar em TODOS os projetos)

### Banco de Dados
- **Soft delete** sempre (`deleted_at` ou `is_active`) — nunca hard-delete em audit trails
- **Tenant isolation** obrigatório em queries multi-tenant
- **RLS** (Row Level Security) em produção
- **`.maybeSingle()`** sempre — nunca `.single()` (evita crash PGRST116)

### Código
- **Conventional Commits**: `feat:`, `fix:`, `docs:`, `refactor:`, `chore:`, `test:` + Co-Authored-By
- **Code review obrigatório** via Buchecha antes de aplicar
- **TypeScript strict** quando possível
- **Zod** para validação de inputs

### Git Workflow Autônomo (padrão obrigatório — ver `GIT-WORKFLOW-AUTONOMO.md`)
- **Claude executa commit + push + verificação pós-deploy sozinho** — Marcelo não digita comandos de git
- **Author obrigatório:** `mfalcao09` / `contato@marcelofalcao.imb.br` (senão Vercel falha)
- **Autenticação centralizada:** Desktop Commander (caminho primário) usa `osxkeychain` nativo do Mac. Fallback sandbox: fine-grained PAT em `/mnt/GitHub/.github-token` + `credential.helper store` global (§2.5)
- **NUNCA** embutir PAT na URL do remote — sempre URL limpa `https://github.com/mfalcao09/<repo>.git`
- **NUNCA** PAT classic — sempre fine-grained com permissões mínimas (`Contents:RW + Metadata:R + PullRequests:RW`)
- **Build completo antes do push** (`next build` ou `npm run build`, não só `tsc`) quando há sessões paralelas
- **`git add` por arquivo**, nunca `-A` / `.` (evita commit de secrets)
- **HEREDOC + Co-Authored-By** em todas as mensagens de commit
- **NUNCA** `--force`, `--amend`, `--no-verify` sem confirmação explícita
- **Pós-push autônomo:** **Vercel CLI** (deploy status — `vercel list` + `vercel inspect --wait` + `vercel logs`) + MCP Sentry (regressões) + MCP Supabase (advisors se migration). MCP Vercel perdeu conexão em 11/04/2026, CLI com `$VERCEL_TOKEN` é o substituto permanente.
- **Troubleshooting FUSE/bindfs:** clone em `/tmp` é o workaround canônico — documentado em `GIT-WORKFLOW-AUTONOMO.md` §6.4

### Segurança
- **NUNCA** expor `service_role_key` no client
- **NUNCA** usar CORS wildcard `*` — sempre whitelist
- **DOMPurify** antes de renderizar HTML de IA

### Memória
- **Auto-save absoluto** — salvar em `memory/` a cada sessão
- **Orient → Work → Persist** — ler memória → trabalhar → salvar
- **Atualizar CENTRAL-MEMORY.md** ao final de cada sessão
- **Inventário obrigatório:** ao criar/instalar qualquer skill, plugin, MCP ou IA → atualizar `INVENTARIO-FERRAMENTAS.md` imediatamente + commit `docs: update inventario — [nome]`
